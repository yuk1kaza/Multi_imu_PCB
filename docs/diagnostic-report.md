### 🔴 Slot 3 - BMI270: **不稳定/硬件问题**
```
第一次运行:
  Probe: r00d1=0x24 (正确!) ✅
  检测: PROBE OK → Init FAILED (ConfigError)

第二次运行:
  Probe: r00d1=0xFF (无响应!) ❌
  检测: probe failed
```

**严重问题:** 传感器状态在两次运行之间变化！

**可能原因:**
1. 🔋 **电源供电不稳定**
   - 建议: 用示波器检查 3.3V 电源纹波
   - 检查: 去耦电容是否正确焊接
   - 测试: 用万用表测量电压稳定性

2. ⏱️ **上电延迟不足**
   - BMI270 可能需要更长的稳定时间
   - 已增加延迟: 500ms → 2000ms

3. 🔌 **焊接质量问题**
   - 检查: 是否有虚焊或冷焊点
   - 检查: 引脚是否短路

4. 🌡️ **传感器故障**
   - 可能是损坏的芯片
   - 建议更换传感器

**需要的操作:**
1. ⚡ **检查电源供电**（最重要！）
   - 测量 VCC 引脚电压（应该是 3.3V ±0.1V）
   - 检查电流供应是否充足
   - 用示波器检查电源纹波

2. 🔬 **检查焊接质量**
   - 用放大镜检查焊点
   - 确认所有引脚都正确连接

3. 🔄 **多次测试**
   ```bash
   # 运行多次看结果是否一致
   cargo r  # 第一次
   cargo r  # 第二次
   cargo r  # 第三次
   ```
   如果每次 probe 结果都不同，肯定是硬件问题

### ⚠️ Slot 4 - QMI8658A: **未知传感器 (CHIP_ID = 0x3E)**
```
Probe: 与 Slot 2 完全相同
  Mode 0: r00=0x3E
  Mode 3: r00=0x00 r01=0x3E
检测: 所有驱动 probe 失败
```

**分析:**
- 与 Slot 2 相同的异常值
- QMI8658A 应该在 r00 返回 0x05，但读到 0x3E
- 两个不同槽位读到相同的异常值很可疑

**可能性:**
1. 两个槽位都焊接了错误的传感器
2. 芯片标记与实际型号不符
3. 山寨/兼容芯片使用不同的 CHIP_ID

**需要的操作:**
1. 🔍 **物理检查 Slot 4 的芯片标记**
2. 🔎 对比 Slot 2 和 Slot 4 是否是相同型号

---

### ❌ Slot 5 - SC7U22: **无响应 (传感器可能未焊接)**
```
Probe: 所有寄存器返回 0xFF
检测: 所有驱动 probe 失败
```

**分析:**
- 0xFF 通常表示总线上拉电阻的默认值
- 说明没有设备响应 SPI 通信

**可能原因:**
1. ❌ 传感器未焊接到 PCB
2. ❌ CS 引脚 (GPIO10) 连接错误
3. ❌ 传感器焊接有虚焊
4. ❌ 传感器已损坏

**需要的操作:**
1. 👀 **目视检查** Slot 5 位置是否有芯片
2. 🔌 **检查连接**
   ```
   用万用表测量:
   - GPIO10 与传感器 CS 引脚的连通性
   - VCC 引脚应该有 3.3V
   - GND 引脚应该接地
   ```
3. 🔬 **检查焊接** 是否所有引脚都正确焊接
